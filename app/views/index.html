1. Naming the service

We have been workshopping names inside the team and with users to name the service. We have settled on the name 'Court cases and release dates'.

2. Addition of an adjustments module on the overview

We've been working with adjustments to agree a way to surface more information relating to adjustments on the overview. This would allow users extra quick glance information to respond to queries, but with an added benefit that user will still be able to see the adjustments even if there is an intercept page on the adjustments tab.

  • Overview - no adjustments to show
  • Overview v1 - summary list - we're using this version
  • Overview v2 - cards

3. Calculations from NOMIS

If a calculation is from NOMIS, there will be no calculation breakdown. Instead we need to signify that the calculation came from NOMIS.

  • Overview - latest calculation is from NOMIS
  • Release dates - latest calculation is from NOMIS
  • calculation breakdown - calculation is from NOMIS

4. Tweaks to the release dates card and release dates tab

We've updated the release dates card design to now surface calculation reason alongside date and establishment. Because of this we have had to add additional headings to the release dates tab which means the calculation history page no longer requires a card design.

5. Updates to the 'things to do' pattern

We have been revising the designs of the 'things to do' notification pattern to try and simplify it and it's content. We've decided to use GOV.UK notification banners instead of a custom component.

We'll be testing this in UR R3 and will be carrying out further design and analysis of notifactions as a whole.

This protoype only includes examples relevant for next UR round, all examples with explainations of logic will be available elsewhere.

  • Overview - ADAs first-time review
  • Release dates - ADAs first-time review
  • Overview - Adjustment added, calculate release dates
  • Release dates - Adjustment added, calculate release dates

6. Updates to the entry points to the service

After multiple sessions with Connect DPS, we've agreed that we will explore ways in which we can use role-based access to customise the prisoner profile to better suit the needs of our users, and improve entry points.

Some of the changes we could propose are:

  • Repurposing the current 'Offences and release dates' card to add relevant information
  • Repositioning the cards on the page
  • Removing the offences tab
  • Accounting for CRDS-only users in the design

Most design options are on the miro board, this prototype only includes the relevant versions for user research.

  • prisoner profile - full-access users
  • prisoner profile - CRDS-only users

7. New footer

Our service requires a specialist support team to help with sentence calculation queries. The current DPS footer points the user to the general support desk for help, this is not suitable for our users.

DPS currently have not have a way to accomodate our support requirements, because of this, we have been given permission to create our own footer to use whilst a strategic solution for support can be implemented by DPS connect.

Versions of the footer can be found on the miro.

1. Updates to navigation to the hub

We've made two key decisions about the navigation to the hub. Vire the full entry points maps on miro.

  • For prisons with CRDS only, there will be no tile on the DPS homepage. The only way into the service will be through the existing 'calculate release dates' link on the prisoner profile.
  • For prisons with at least CRDS and ADJ, we will have a new link to the hub overview on the prisoner profile under the 'more information' cards instead of the 'actions' card. We will have no actions when we intially roll-out - the entry point on DPS homepage will still maintain action tiles. We will monitor this in beta and add any actions links where necessary.

2. Updates to the overview

Removing tiles

We've been reconsidering the three dashboard tiles at the top of the page which don't provide a lot of function for their size.

'Things to do' notifications

We've been looking at cross-service notifications. Off the back of this work, we have added the 'Things to do' section back in to explore this further. Initially we envisage things like reviewing ADAs and prompts to calculate release dates to be shown here. Some examples:

  • All examples (miro)
  • Review ADAs
  • Review PADAs
  • Calculate release dates

3. Intercept pages across the service

In some scenarios, we may need the user to do something before they can carry out certain actions on the service. An example of this is where there are ADAs to review, the user should not be able to add another adjustment or calculate release dates before reviewing the ADAs.

  • Overview tab - ADAs to review
  • Adjustments tab - ADAs to review
  • Calculate release dates tab - ADAs to review
  • Calculate release dates journey - ADAs to review

5. CRDS improvements

We've been looking at ways we need to amend CRDS journeys in order to fit the hub features in, such as removing the start page. Alongside this, we've been making some improvements to existing journeys along the way.

Surfacing calculation reason

CRDS now captures a reason for calculation, we now surface this when the user looks up a calculation.

Calculation history Experimental

We've been looking to improve the calculation look-up journey. Now that there is no start page, we surface all previous calculation in a table on the calculations and release dates tab.

Because some calculations may come from NOMIS, we've had to consider some of the imprications of this on our design. One of these being that NOMIS automatically produces a new calculation any time that there is a change to a prisoners record. This could muddy up our table. (DESIGN TBC)

We've also improved the user experience of viewing the details of an existing calculation. Previously this replicated the calculation journey and was always seen as a temporary journey.

Some new calculation details options below:

  • Side nav, condensed onto one page
  • Side nav, separate pages
  • Sub nav (WIP)
  • Accordion (WIP)

6. Read-only court cases tab Experimental

We've previously assumed that before R&S is released, users would not see the court cases tab (see version 5). We're now considering wether we could use data from NOMIS in order to provide the user with a read-only version of the court cases tab earlier. This is subject to agreement with R&S team and designs would be lead by them, for now we're just experimenting for the sake of upcoming UR.
