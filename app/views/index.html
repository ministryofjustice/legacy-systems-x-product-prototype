What we are testing

This remote round of UR is due to take place in wc 29th January 2023.

In this round, we aim to:

  • Understand how hub requirements might differ across prison types (we have focused on local prisons to date).
  • Identify specific requirements for service features that users have told us are important to them (e.g.overview page, documents, calculation history).
  • Observe how users interact with the "hub" in a transfer check scenario.
  • Test the options for service name that came out of naming workshop and assessment.

3. CRDS improvements

We've been looking at ways we need to amend CRDS journeys in order to fit the hub features in, such as removing the start page. Alongside this, we've been making some improvements to existing journeys along the way.

Surfacing calculation reason

CRDS now captures a reason for calculation, we now surface this when the user looks up a calculation.

Calculation history

We've been looking to improve the calculation look-up journey. Now that there is no start page, we surface all previous calculation in a table on the calculations and release dates tab.

Because some calculations may come from NOMIS, we've had to consider some of the imprications of this on our design. One of these being that NOMIS automatically produces a new calculation any time that there is a change to a prisoners record. This could muddy up our table. (DESIGN TBC)

We've also improved the user experience of viewing the details of an existing calculation. Previously this replicated the calculation journey and was always seen as a temporary journey.

Some new calculation details options below:

  • Side nav, condensed onto one page
  • Side nav, separate pages
  • Sub nav (WIP)
  • Accordion (WIP)

4. Read-only court cases tab

We've previously assumed that before R&S is released, users would not see the court cases tab (see version 5). We're now considering wether we could use data from NOMIS in order to provide the user with a read-only version of the court cases tab earlier. This is subject to agreement with R&S team and designs would be lead by them, for now we're just experimenting for the sake of upcoming UR.

5. Snapshots of what the hub looks like at each stage of release

We need to consider what the hub might look like a different stages of release.

Adjustments will be rolled out around March but not to all prisons currently using the CRDS service, this might be the same scenario when it comes time to release Remand and Sentencing.

This means that there may be multiple variations of the hub in different prisons at one time. We need to account for this in the design.

This is the first version of the hub design during this new phase of work.
